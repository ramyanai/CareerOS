# /gaps — Skills Gap Aggregation Report

You are a career analyst. Your job is to aggregate keyword gap data from all strategy documents and produce a prioritized skills gap report showing which skills to develop first.

## Step 1: Find All Strategy Documents

Use Glob to find all strategy docs:
```
~/CareerOS/output/*/_Strategy_.md
~/CareerOS/output/*/_application_strategy.md
~/CareerOS/output/*/application_strategy.md
```

Also try `*Strategy*` and `*strategy*` patterns to catch all naming conventions.

If no strategy docs are found, tell the user to run `/apply` first and stop.

## Step 2: Parse Keyword Gap Tables

For each strategy doc found:

1. Read the file
2. Find the **Keyword Gap Analysis** table (Section 1). It has columns: `JD Keyword / Phrase | Found in Resume? | Action Taken`
3. Extract the company name from the document title (first `# Application Strategy: {COMPANY_NAME}` heading)
4. For each row in the table, classify the keyword into one of three categories:

**Classification rules:**
- **Strength** (green): "Found in Resume?" is `Yes` or `Already present`, OR "Action Taken" says `Already present` / `Already in resume`
- **Addressed** (yellow): "Found in Resume?" is `Partial` or `Reframed`, OR "Action Taken" says `Reframed` / `Added` / `Partial` / `Elevated` / `Emphasized`
- **Gap** (red): "Found in Resume?" is `No`, OR "Action Taken" contains `Cannot add` / `Gap` / `Not in experience` / `not explicitly`

When a keyword appears in multiple applications, track its classification per company.

## Step 3: Aggregate Results

Build three aggregate tables:

### Strengths (consistently matched)
Keywords classified as **Strength** across applications. Sort by frequency (most common first).

### Priority Gaps (upskilling targets)
Keywords classified as **Gap** in any application. Sort by frequency (most gaps first). For each gap, suggest a concrete action the user could take to fill it.

**Suggested actions for common gap types:**
- Speaking/presenting → "Give a talk at a local meetup, start a YouTube channel, or join Toastmasters"
- Specific technology → "Build a side project using X, complete a certification"
- Domain experience → "Contribute to open-source projects in this space, write blog posts"
- Management/leadership → "Mentor juniors, lead a community project, volunteer for team lead roles"
- Industry knowledge → "Read industry publications, attend conferences, network with practitioners"

### Full Keyword Matrix
A matrix with keywords as rows and companies as columns. Each cell shows: `Strength`, `Addressed`, `Gap`, or `N/A` (keyword not in that JD).

## Step 4: Generate Recommendations

Write top 3 prioritized recommendations based on:
1. **Frequency** — skills that appear as gaps in the most applications
2. **Addressability** — skills the user could realistically develop (vs. requiring years of experience)
3. **Impact** — skills that would unlock entire categories of roles

## Step 5: Write Report

Write the report to `~/CareerOS/output/gaps_report_{YYMMDD}.md` where YYMMDD is today's date.

Use this format:

```markdown

## Full Keyword Matrix

| Skill | Anthropic | Amazon | ... |
|-------|-----------|--------|-----|
| Developer evangelism | Addressed | N/A | ... |
```

Run:
```bash

```
